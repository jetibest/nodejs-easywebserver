Welcome to Easywebserver

Running on Node.js

Easywebserver is basically a wrapper for Express.js, and makes running a webserver a one-liner. Flexibility is provided by modules. These mods are included by default:

  • [pre-route] mod-dirtofile Match extension based on directory-based pathname. For instance, /page/ may internally route to /page.html.
  • [pre-route] mod-forcedir Ensures a trailing slash in the pathname using a redirect (302). For instance, /page may redirect to /page/.
  • [pre-route] mod-hide Blocks access to directories or files based on a regular expression. Defaults to blocking files and directories starting with ., _, and $.
  • [pre-route] mod-urlrewrite Generic regular expression-based URL rewriting for internal routing only (no redirects, meaning the rewriting is not visible for clients i.e. in the address bar).
  • [catch-extension] mod-php Enable delivery of PHP-files (.php) using php-fpm. This mod will run its own php-fpm daemon.
  • [catch-all] mod-html Enable delivery of static files such as .html, .css, .js, .jpg, etc. in the public_html directory.
  • [error] mod-reroute Reroute the request to a different path, given a match is found. This is an internal redirect, sending the Request and Response object back to square one (to the first middleware handler of the app). By default reroutes to /http-404.html.
  • [post-process] mod-log Basic logging to stdout.

For better documentation, check out the Readme.md on github.com.
